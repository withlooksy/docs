## Make Looksy Match Your Brand

Looksy is designed to integrate seamlessly with your store's aesthetic. Customize colors, fonts, sizing, and more to create a cohesive brand experience.

## Theme Editor Customization (No Code)

### Basic Settings

Available in the Shopify theme editor when you click on the Looksy block:

<AccordionGroup>
  <Accordion title="Button Text">
    **Default:** "Try On"

    **Alternatives:**
    - "See it on you"
    - "Virtual try-on"
    - "Preview on yourself"
    - "Try on virtually"
    - Custom text of your choice

    **Best practice:** Keep it short (2-4 words) and action-oriented.
  </Accordion>

  <Accordion title="Button Color">
    **Options:**
    - Use theme primary color (default)
    - Use theme secondary color
    - Custom hex color (#0D9373)

    **Best practice:** Choose a color that contrasts with your background for visibility.
  </Accordion>

  <Accordion title="Button Size">
    **Options:**
    - Small
    - Medium (recommended)
    - Large

    **Best practice:** Match the size of your "Add to Cart" button for consistency.
  </Accordion>

  <Accordion title="Alignment">
    **Options:**
    - Left
    - Center
    - Right

    **Best practice:** Match the alignment of other buttons on your product page.
  </Accordion>
</AccordionGroup>

## CSS Customization (Advanced)

For more control, add custom CSS to your theme.

### Accessing Theme CSS

1. Go to **Online Store → Themes**
2. Click **"Actions" → "Edit code"**
3. Find **Assets → theme.css** (or custom CSS file)
4. Add your custom styles at the bottom

### Button Styling

#### Basic Button Customization

```css
/* Primary button styling */
.looksy-try-on-button {
  background-color: #0D9373 !important;
  color: white !important;
  font-size: 16px !important;
  font-weight: 600 !important;
  padding: 12px 24px !important;
  border-radius: 8px !important;
  border: none !important;
  cursor: pointer !important;
  transition: all 0.3s ease !important;
}

/* Hover state */
.looksy-try-on-button:hover {
  background-color: #0A7A5E !important;
  transform: translateY(-2px) !important;
  box-shadow: 0 4px 8px rgba(0, 0, 0, 0.15) !important;
}

/* Active/pressed state */
.looksy-try-on-button:active {
  transform: translateY(0) !important;
}
```

#### Outlined Button Style

```css
.looksy-try-on-button {
  background-color: transparent !important;
  color: #0D9373 !important;
  border: 2px solid #0D9373 !important;
  font-weight: 600 !important;
  padding: 10px 22px !important;
  border-radius: 6px !important;
}

.looksy-try-on-button:hover {
  background-color: #0D9373 !important;
  color: white !important;
}
```

#### Rounded Button Style

```css

  border-radius: 50px !important;
  padding: 14px 32px !important;

```

#### Minimalist Button Style

```css
.looksy-try-on-button {
  background-color: #f5f5f5 !important;
  color: #333 !important;
  border: 1px solid #ddd !important;
  border-radius: 4px !important;
  padding: 12px 20px !important;
  font-size: 14px !important;
  text-transform: uppercase !important;
  letter-spacing: 0.5px !important;
}

  background-color: #e8e8e8 !important;
}
```

<Tip>
Use `!important` to ensure your styles override theme defaults. While not ideal in general CSS, it's necessary for reliable customization of app components.
</Tip>

## Responsive Styling

### Mobile-Specific Styles

```css
/* Desktop styles */
.looksy-try-on-button {
  padding: 12px 24px !important;
  font-size: 16px !important;
}

/* Mobile styles */
@media (max-width: 768px) {
  .looksy-try-on-button {
    padding: 14px 20px !important;
    font-size: 14px !important;
    width: 100% !important;
  }

  /* Sticky mobile button */
  .looksy-try-on-button.sticky {
    position: fixed !important;
    bottom: 0 !important;
    left: 0 !important;
    right: 0 !important;
    border-radius: 0 !important;
    z-index: 999 !important;
  }
}

/* Tablet styles */
@media (min-width: 769px) and (max-width: 1024px) {
  .looksy-try-on-button {
    padding: 12px 20px !important;
    font-size: 15px !important;
  }
}
```

## Dark Mode Support

If your theme supports dark mode:

```css

  background-color: #0D9373 !important;
  color: white !important;
}

/* Dark mode */
@media (prefers-color-scheme: dark) {
  .looksy-try-on-button {
    background-color: #07C983 !important;
    color: #1a1a1a !important;
  }

  .looksy-modal {
    background-color: #1a1a1a !important;
    color: white !important;
  }
}

/* Theme-specific dark mode class */
.dark-mode .looksy-try-on-button {
  background-color: #07C983 !important;
}
```


## Common Styling Issues

<AccordionGroup>
  <Accordion title="My styles aren't being applied">
    **Possible causes:**
    - Missing `!important` flag
    - Incorrect CSS selector
    - Theme CSS loaded after your custom CSS

    **Solutions:**
    1. Add `!important` to your styles
    2. Use browser dev tools to inspect correct selector
    3. Add styles to theme.css or custom CSS file at the end
  </Accordion>

  <Accordion title="Button looks different on mobile">
    **Possible causes:**
    - No mobile-specific styles defined
    - Theme has mobile overrides

    **Solutions:**
    1. Add `@media` queries for mobile
    2. Check theme's mobile CSS
    3. Use `!important` to override theme styles
  </Accordion>

  <Accordion title="Hover effects don't work on mobile">
    **Expected behavior:** Touch devices don't have hover states

    **Solutions:**
    Use `:active` state for touch feedback:
    ```css
    .looksy-try-on-button:active {
      background-color: #0A7A5E !important;
    }
    ```
  </Accordion>

  <Accordion title="Custom font doesn't load">
    **Possible causes:**
    - Font not loaded in theme
    - Incorrect font-family name

    **Solutions:**
    1. Ensure font is loaded in theme settings
    2. Use exact font-family name from theme
    3. Provide fallback fonts
  </Accordion>
</AccordionGroup>

## CSS Variables for Easy Maintenance

Set up CSS variables for easier updates:

```css
:root {
  --looksy-primary: #0D9373;
  --looksy-hover: #0A7A5E;
  --looksy-text: white;
  --looksy-radius: 8px;
  --looksy-padding: 12px 24px;
}

.looksy-try-on-button {
  background-color: var(--looksy-primary) !important;
  color: var(--looksy-text) !important;
  border-radius: var(--looksy-radius) !important;
  padding: var(--looksy-padding) !important;
}

.looksy-try-on-button:hover {
  background-color: var(--looksy-hover) !important;
}
```

**Benefits:** Change one variable to update all instances.

## Performance Considerations

### Keep It Lightweight

- Avoid heavy animations (impacts performance)
- Don't add large background images to buttons
- Use CSS transforms instead of position changes
- Minimize use of box-shadows

### Optimize for Speed

```css
/* Use transform instead of top/left for animations */
.looksy-try-on-button:hover {
  transform: translateY(-2px) !important; /* Better performance */
  /* top: -2px; -- Avoid this */
}

/* Use will-change for animated elements */
.looksy-try-on-button {
  will-change: transform !important;
}
```

## Accessibility Considerations

### Ensure Good Contrast

Use tools like WebAIM Contrast Checker to verify:
- Button text is readable (4.5:1 contrast minimum)
- Focus states are visible
- Color isn't the only indicator of interactivity

```css
/* Visible focus state for keyboard navigation */
.looksy-try-on-button:focus {
  outline: 3px solid #0D9373 !important;
  outline-offset: 2px !important;
}
```
